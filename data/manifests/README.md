# Evaluation manifests

Generated by `harness/make_manifests.py` (deterministic; safe to re-run).
Loading/parsing logic lives in `harness/datasets.py`.

Common record fields: `problem_id`, `source` (`workbook` | `minif2f_valid`),
`difficulty`, `theorem_name`. Workbook records additionally carry the verbatim
`imports`, `preamble`, `statement`, and `rocq_proved` (whether the source
dataset ships a checked proof — informational only). miniF2F records carry
`source_tier`, `has_shipped_proof`, `proof_status`, and a `path` relative to
the directory containing the dataset checkouts (content stays on disk).

## Manifests

| file | size | contents |
|---|---|---|
| `dev60.jsonl` | 60 | workbook, `rocq_library=="stdlib"` only, 20 per difficulty (easy/medium/hard) |
| `dev150.jsonl` | 150 | same pools, 50 per difficulty, **disjoint from dev60** |
| `smoke5.jsonl` | 5 | 5 easy stdlib problems for debugging; **may overlap** dev60/dev150 |
| `minif2f_valid.jsonl` | 244 | all miniF2F-rocq *valid* problems |

## Sampling procedure

- Pool: workbook records with `rocq_library == "stdlib"`, grouped by
  difficulty and **sorted by `name`** (determinism does not depend on jsonl
  line order). Pool sizes: easy 1593, medium 5687, hard 1656.
- One generator `random.Random(42)`; for each difficulty in order
  (easy, medium, hard) draw `rng.sample(pool, 70)` in a single pass:
  the first 20 picks form dev60, the next 50 form dev150. dev60 and dev150
  are therefore disjoint by construction (verified: intersection = 0).
- `smoke5` = `random.Random(7).sample(easy_pool, 5)` (first 5 of a seed-7
  sample). Debug only; overlap with dev60/dev150 is allowed.

To materialize a workbook problem as a `.v` file, use
`harness/datasets.workbook_problem_to_vfile(rec)`, which replicates the
assembly of `rocq-workbook/provenance/materialize.py` exactly
(imports / optional preamble / blank line / statement) minus the header
comment and proof block. Append `\nAdmitted.\n` to type-check a bare
statement (spot-checked: compiles with `rocq compile`, exit 0).

## miniF2F difficulty proxy (source tiers)

miniF2F has no difficulty labels; we use the competition source parsed from
the filename prefix as a proxy. Raw prefixes are normalized
(`amc12a_`/`amc12b_`/`amc12_` → `amc12`; `aime_`/`aimeI_`/`aimeII_` → `aime`);
prefixes without a competition source (`algebra_`, `numbertheory_`,
`induction_`) are the MiniF2F custom problems.

| source_tier | valid | test |
|---|---|---|
| mathd_algebra | 70 | 70 |
| mathd_numbertheory | 60 | 60 |
| amc12 | 45 | 45 |
| imo | 20 | 19 |
| imosl (IMO shortlist) | 0 | 1 |
| aime | 15 | 15 |
| algebra | 18 | 18 |
| numbertheory | 8 | 8 |
| induction | 8 | 8 |
| **total** | **244** | **244** |

**Held-out discipline:** the *test* split is locked. The test-column counts
above were obtained from **filenames only** (`ls` on the test directory); no
test file contents were read. The only code path allowed to read test file
contents is the guarded branch in `harness/datasets.load_minif2f("test")`,
which requires both `<repo>/FINAL_UNLOCK` and `ROCQ_FINAL_EVAL=1` and logs
every unlock to `<repo>/logs/unlock.log`.

## Known dataset irregularities

- All 244 valid miniF2F files end in either `Proof.` + `Admitted.`
  (149 files) or a **bare trailing `Proof.`** with no `Admitted`/`Qed`
  (95 files). No valid file ships a completed proof
  (`has_shipped_proof` is `false` everywhere; see `proof_status`).
- `miniF2F-rocq/valid/imo_1966_p5.v` contains **two** theorems; the target
  (last) theorem is named `imo_1966_p5'` (trailing apostrophe), which does
  not match the filename stem. `theorem_name` in the manifest reflects the
  actual identifier.
- 25 valid miniF2F files carry helper `Definition`/`Fixpoint` code before
  the target theorem; `harness/datasets.statement_prefix` keeps such
  helpers in the prefix.
